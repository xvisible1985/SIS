Как выглядит flow создания бота?

После клика «Создать бота» пользователь проходит через несколько шагов. Какой вариант лучше?

A

Пикер → Единая форма

Шаг 1: модалка выбора типа (уже есть). Шаг 2: открывается та же BotForm, но с пред-выбранным типом и соответствующими дефолтами. Тип можно сменить внутри формы.

Выбор типа → Общая форма (адаптирована)
  • Минимум новых компонентов
  • Быстро
  • Форма «пытается» охватить всё
B

Пикер → Специализированная форма

Шаг 1: модалка выбора типа. Шаг 2: открывается своя форма под каждый тип — TrendBotForm, ParserBotForm, HedgeBotForm. Каждая показывает только нужные поля.

Выбор типа → TrendForm / ParserForm / HedgeForm
  • Чистый UX, нет лишних полей
  • Легко добавлять типы
  • Дублирование общих полей (name, symbol…)
C

Всё в одном окне (таб-пикер + форма)

Одна большая модалка: слева или сверху — табы с типами (иконки), справа — форма меняется при переключении. Выбор типа и заполнение — в одном месте.

Тип-таб | Форма → Создать
  • Один шаг вместо двух
  • Легко сравнить типы
  • Сложнее в реализации
  • Окно большое
